# EZRA 5.0 — מדריך שימוש ליזם

## מה יש בתיקייה הזו?

| קובץ | תפקיד |
|------|--------|
| `template.html` | תבנית הדוח — לא לשנות |
| `prompt.md` | הפרומפט לכל AI |
| `instructions.md` | המדריך הזה |

---

## איך מייצרים דוח — 3 צעדים

### צעד 1 — הכן את הפרומפט
פתח את `prompt.md` והעתק את כל התוכן שלו.

### צעד 2 — צור JSON
1. פתח כל AI: Claude / ChatGPT / Gemini (גם גרסאות חינמיות עובדות)
2. הדבק את הפרומפט
3. הוסף מיד אחריו את מסמך התפיסה שלך
4. שלח — קבל JSON

### צעד 3 — הרץ את הדוח
1. פתח את `template.html` בעורך טקסט (Notepad, VS Code, כל עורך)
2. מצא את השורה:
   ```
   const REPORT_DATA = {
   ```
3. מחק את כל ה-JSON שבתוכה (מ-`{` הפותח עד `}` הסוגר)
4. הדבק את ה-JSON החדש שקיבלת מה-AI
5. שמור את הקובץ
6. פתח את `template.html` בדפדפן — הדוח מוכן!

---

## ייצוא הדוח

**PDF:** לחץ על כפתור "📄 ייצוא ל-PDF" → בדפדפן בחר "שמור כ-PDF"

**HTML לשיתוף:** לחץ על "💾 שמור HTML" — מקבל קובץ HTML שאפשר לשלוח לכל אחד

**עריכה ידנית:** לחץ "🎨 התחל עריכה" — ניתן לערוך כל טקסט ישירות בדפדפן

---

## פתרון בעיות

**הדוח ריק / שגיאה בדפדפן?**
→ JSON לא תקני. בדוק שאין תווים מיוחדים שבורים. נסה להכניס את ה-JSON ל-[jsonlint.com](https://jsonlint.com) לבדיקה.

**ה-AI נתן טקסט ולא JSON?**
→ וודא שהדבקת את כל הפרומפט כולל ה"כעת נתח..." בסוף.

**העיצוב שבור?**
→ לא לשנות כלום ב-`template.html` מלבד ה-`REPORT_DATA`.

## תחזוקה ועדכונים

כשיש עדכון ל-EZRA:
- תקבל קובץ `template.html` חדש בלבד — להחליף
- ה-`prompt.md` מתעדכן בנפרד
- ה-JSON שייצרת תמיד תואם לתבנית
